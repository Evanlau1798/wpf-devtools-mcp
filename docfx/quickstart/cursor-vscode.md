# Cursor and VS Code Setup

Cursor and VS Code are easiest to configure from the JSON artifact generated by the installer.

## Installed executable path

```text
%LOCALAPPDATA%\WpfDevToolsMcp\x64\current\WpfDevTools.Mcp.Server.exe
```

## Generated JSON template

The installer writes `client-registration\cursor-vscode.json`. Its structure is:

```json
{
  "servers": {
    "wpf-devtools": {
      "command": "%LOCALAPPDATA%\\WpfDevToolsMcp\\x64\\current\\WpfDevTools.Mcp.Server.exe",
      "args": []
    }
  }
}
```

Use the same command path in Cursor or `.vscode/mcp.json`.

## First useful workflow

1. Ask the client to call `tools/list`.
2. Run `get_processes`.
3. Run `connect`.
4. Run `get_visual_tree`.

## Notes

- Re-register the installed executable after switching architectures.
- Keep editor-side wrappers from writing to `stdout`.
